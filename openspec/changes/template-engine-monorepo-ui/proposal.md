## Why

Provide an interactive web-based playground for the template engine kata to practice Hexagonal Architecture and frontend React guidelines in a monorepo structure. This allows users to input templates and variables dynamically, processing them through a backend parser that implements the Notifier pattern to gather precise warning/info events about the parsing execution.

## What Changes

- **Workspace Reorganization**: Structure the workspace as a monorepo containing `apps/backend` (Express/TypeScript) and `apps/web` (Vite/React/TypeScript).
- **Domain Refactoring**: Upgrade the core template engine to return a rich result (`SUCCESS`, `PARTIAL`, `FAILED`), the parsed text, and structured notifications about variable replacements, null entries, and missing or unused placeholders.
- **Backend API**: Build an Express API server in `apps/backend` exposing the template parsing engine as an HTTP endpoint.
- **React Frontend**: Implement a responsive web application in `apps/web` using Vite, pure CSS Modules, and strict React/custom hook standards. This includes:
  - An interactive dynamic key-value variables table (Option C).
  - A results card with visual parsing status badges.
  - A real-time system logs console displaying notifications generated by the domain parser.

## Capabilities

### New Capabilities

- `template-parsing-engine`: A zero-dependency core domain parser in the backend implementing the rich Notifier pattern and exposed via Express HTTP API.
- `template-engine-ui`: A modern web client built with React + Vite + CSS Modules in the frontend, adhering to custom hook and composition rules.

### Modified Capabilities

None.

## Impact

- Moves existing `src/core/template-engine.ts` and tests into the backend package (`apps/backend`).
- Sets up monorepo package configurations in root.
- Exposes an HTTP API for template processing.
- Launches Vite development server for frontend.
